system design life cycle

1. requirements

Define performance, cost, power, and reliability goals. workload characterization & interface contracts.

2. high‑level

Partitioning, major blocks (cores, memory hierarchy, I/O). chiplet or monolithic? topology exploration.

3. microarchitecture

Pipeline stages, cache design, arbitration, buffering. state machines and dataflow.

4. verification

Formal property checks, simulation, performance validation against spec. sign-off.

example: 8‑core mesh system

  • 8 × out‑of‑order cores (RISC‑V ISA)
  • private L1 / L2 per core cluster
  • distributed shared L3 (mesh‑connected)
  • DDR5 controllers at mesh edges
  • coherent mesh fabric (MESI protocol)
  • PCIe 5.0 root complex & IOMMU
estimated specs: 3.2 GHz, 64 MB L3, 512 GB/s socket bandwidth

core principles

01

separation of concerns

Decouple policy from mechanism. Keep interfaces stable, internals free to optimize.

02

scalable fabric

Design interconnect and coherence to scale from 2 to 100+ cores with predictable latency.

03

design for verifiability

Add observability, assertions, and avoid ad‑hoc state machines.

04

power‑performance pareto

Model energy at each stage; clock gating, DVFS, and memory hierarchy tuning.
